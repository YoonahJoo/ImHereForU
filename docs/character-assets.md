# 캐릭터 에셋 요청서

GPT에게 이미지 생성을 요청할 때 사용하는 에셋 명세입니다.

---

## 공통 조건 (모든 파일 동일 적용)

- 캔버스 크기: **1110 × 1417px**
- 배경: **완전 투명 PNG**
- 해당 레이어 부위 외 나머지 영역은 완전 투명
- 캐릭터 스타일, 색상, 비율은 첫 번째 생성 이미지를 레퍼런스로 고정
- 눈 영역은 face 이미지에서 비워두고 eyes 레이어로 별도 처리

---

## 에셋 목록 (총 15개)

### 1. body (1개)

| 파일명 | 내용 |
|---|---|
| `body.png` | 몸통 전체. 눈 영역과 팔 영역은 투명하게 비워둠 |

---

### 2. face (10개)

눈 영역은 비워두고 입 모양, 눈썹, 볼터치 등 눈 이외의 표정 요소만 포함.

| 파일명 | 표정 | 설명 |
|---|---|---|
| `face_idle.png` | 기본 | 평온한 입 모양 |
| `face_smile.png` | 미소 | 눈웃음 느낌의 눈썹, 볼터치, 올라간 입꼬리 |
| `face_sulky_daily.png` | 화남 | 찡그린 눈썹, 뾰족하게 다문 입 |
| `face_sulky_focus.png` | 코웃음 | 삐딱한 입, 콧방귀 느낌 |
| `face_sleepy.png` | 졸림 | 처진 눈썹, 힘없이 벌어진 입 |
| `face_focus.png` | 집중 | 진지하게 다문 입, 일자 눈썹 |
| `face_dragging_daily.png` | 의심 | 한쪽 눈썹만 올린 표정, 삐죽한 입 |
| `face_dragging_focus.png` | 무표정 | 일자 입, 평평한 눈썹 |
| `face_curious.png` | 호기심 | 동그란 안경 착용, 살짝 갸웃한 눈썹 |
| `face_cheering.png` | 뿌듯함 | 흐뭇하게 올라간 입꼬리, 여유로운 눈썹 |

---

### 3. eyes (2개)

눈 부위만 포함. body 및 face 레이어 위에 겹쳐서 사용.

| 파일명 | 내용 |
|---|---|
| `eyes_open.png` | 눈 뜬 상태 (기본) |
| `eyes_blink.png` | 눈 감은 상태 (깜빡임 애니메이션용) |

---

### 4. arm (2개)

팔 부위만 포함. body 레이어와 별도로 분리.

| 파일명 | 내용 |
|---|---|
| `arm_normal.png` | 기본 팔 위치 (자연스럽게 내린 상태) |
| `arm_wave.png` | 손 흔드는 팔 위치 (한 팔 들어 올린 상태) |

---

## 레이어 적층 순서

```
arm_normal / arm_wave     ← 최하단 (뒤에서 흔듦)
body
face_*                    ← expression 상태에 따라 교체
eyes_open / eyes_blink    ← 주기적으로 교체 (깜빡임)
```

---

## GPT 프롬프트 작성 팁

1. 첫 번째로 `body.png`를 생성해 캐릭터 스타일 확정
2. 이후 모든 파일 요청 시 첫 번째 이미지를 레퍼런스로 첨부
3. 각 요청마다 "동일한 캔버스 크기, 동일한 캐릭터 비율" 명시
4. 눈 영역 투명 처리가 필요한 face 파일은 "눈 위치에 해당하는 영역을 투명하게 비워달라"고 명시
