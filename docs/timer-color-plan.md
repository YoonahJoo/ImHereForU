# Focus 타이머 색상 리디자인 기획안

> 목표: Focus 타이머(타이머 본체 + 타이머 창)의 전체 색상을 프로젝트의 통일된 색감에 맞춘다.

---

## 1. 현황 분석

### 1-1. 타이머가 사용 중인 색상

| 위치 | 요소 | 현재 색상 | 계열 |
|---|---|---|---|
| `FocusTimer.css` | `.focus-timer` 테두리 | `rgba(170,140,210,0.35)` | 보라 |
| `FocusTimer.css` | `.timer-display` 글자 | `#c9849e` | 로즈핑크 |
| `FocusTimer.css` | `.timer-btn` 배경 | `#c9849e` → hover `#a87090` | 로즈핑크 |
| `FocusTimer.css` | `.timer-btn--reset` 배경 | `rgba(180,155,210,0.6)` → hover `rgba(150,120,190,0.85)` | 보라 |
| `YoonahRoom.css` | `.focus-timer-window` 배경 | `rgba(255,245,250,0.96)` | 핑크 화이트 |
| `YoonahRoom.css` | `.focus-timer-window` 테두리 | `rgba(170,140,210,0.35)` | 보라 |
| `YoonahRoom.css` | `.focus-timer-window-title` | `#a87090` | 모브 |
| `YoonahRoom.css` | `.focus-timer-window-close` | `#a87090` | 모브 |

### 1-2. 프로젝트 전반의 색상 팔레트 (사용 빈도 기준)

| 색상 | 빈도 | 역할 |
|---|---|---|
| `#a87090` | 16회 | **주요 텍스트 액센트 (모브/로즈)** |
| `#c8a8d8` | 8회 | 다크 테마 텍스트 (라벤더) |
| `#c9849e` | 7회 | 로즈핑크 액센트 |
| `#9878a8` | 4회 | 뮤트 퍼플 |
| `#5a3a6e` | 3회 | 다크 테마 버튼 (딥퍼플) |
| `#1a1a2e → #16213e → #0f3460` | 다크 배경 그라데이션 | Settings·GiftRoom 공통 |

→ 프로젝트의 정체성 색은 **모브/로즈핑크 계열(`#a87090`, `#c9849e`) + 라벤더/퍼플 계열(`#c8a8d8`, `#5a3a6e`)** 이다.

---

## 2. 문제점

1. **액센트 색이 두 갈래로 섞임** — 같은 타이머 안에서 시작/일시정지 버튼은 로즈핑크(`#c9849e`), 리셋 버튼·테두리는 보라(`rgba(180,155,210,…)`)로 계열이 갈린다. 한 컴포넌트 안에서 메인 액센트가 통일되지 않아 어수선하게 보인다.
2. **다크 테마 대응 누락** — Settings·GiftRoom은 `.theme-dark` 오버라이드로 짙은 네이비 배경에 라벤더 텍스트를 쓰는데, 타이머 본체·타이머 창에는 다크 오버라이드가 전혀 없다. 다크 모드에서 핑크 화이트 카드가 네이비 배경 위에 떠 이질적이다.
3. **하드코딩 분산** — 색상이 두 파일(`FocusTimer.css`, `YoonahRoom.css`)에 흩어져 있어 톤을 한 번에 바꾸기 어렵다.

---

## 3. 색상 설계안

프로젝트의 주 액센트인 **로즈/모브 계열로 액센트를 통일**하고, 보라색은 보조(서브 버튼)로만 은은하게 남긴다. 다크 테마는 기존 라벤더/딥퍼플 규칙을 그대로 따른다.

### 3-1. 라이트 테마 (기본)

| 토큰(의미) | 적용 대상 | 제안 색상 | 비고 |
|---|---|---|---|
| 창 배경 | `.focus-timer-window` | `rgba(255,245,250,0.96)` | 유지 |
| 액센트(메인) | `.timer-display`, `.timer-btn` | `#c9849e` | 유지 |
| 액센트(강조/hover) | `.timer-btn:hover`, 타이틀·닫기 | `#a87090` | 유지 |
| 서브 버튼(리셋) | `.timer-btn--reset` | `rgba(201,132,158,0.5)` → hover `rgba(168,112,144,0.75)` | **보라 → 로즈 톤으로 변경** |
| 테두리 | `.focus-timer`, `.focus-timer-window` | `rgba(201,132,158,0.3)` | **보라 → 로즈 톤으로 변경** |

→ 핵심 변경: **리셋 버튼과 테두리의 보라색을 로즈 계열로 교체**해 타이머 전체를 한 가지 액센트로 통일.

### 3-2. 다크 테마 (신규 추가)

Settings/GiftRoom과 동일 규칙 적용. `.yoonah-room.theme-dark` 하위 선택자로 추가.

| 적용 대상 | 제안 색상 |
|---|---|
| 창 배경 | `linear-gradient(160deg, #1a1a2e 0%, #16213e 55%, #0f3460 100%)` |
| 창 테두리 | `rgba(160,120,220,0.35)` |
| 타이틀·닫기 글자 | `#c8a8d8` (라벤더) |
| 타이머 본체 배경 | `rgba(255,255,255,0.07)` |
| 타이머 본체 테두리 | `rgba(160,120,220,0.35)` |
| 시간 글자(`.timer-display`) | `#c8a8d8` |
| 메인 버튼(`.timer-btn`) | `#5a3a6e` → hover `#7a4a8e` |
| 리셋 버튼 | `rgba(200,168,216,0.25)` → hover `rgba(200,168,216,0.4)` |

## 5. 작업 범위

수정 파일은 **CSS 2개뿐**, 컴포넌트 로직(`.tsx`)·DOM 구조 변경 없음.

- `src/components/FocusMode/FocusTimer.css`
  - 리셋 버튼·테두리 색 로즈 톤으로 교체
  - `.theme-dark` 하위 다크 오버라이드 추가 (타이머 창이 `.yoonah-room.theme-dark` 안에 있으므로 `.theme-dark .focus-timer …` 형태로 작성)
- `src/components/YoonahRoom/YoonahRoom.css`
  - 타이머 창 테두리 로즈 톤으로 교체
  - `.yoonah-room.theme-dark .focus-timer-window …` 다크 오버라이드 추가

### (선택) 유지보수 개선
색상을 `.yoonah-room` 스코프의 CSS 커스텀 프로퍼티(`--timer-accent`, `--timer-accent-strong`, `--timer-surface` 등)로 추출하면 라이트/다크 전환과 향후 톤 조정이 한 곳에서 가능. 본 기획 범위에선 선택 사항으로 두고, 1차는 직접 색상 교체로 진행.

---

## 6. 검증 방법

1. 라이트 모드: 타이머 창 열기 → 시작/일시정지/리셋 버튼·테두리가 모두 로즈 계열로 통일됐는지 확인.
2. 다크 모드(Settings에서 dark 선택): 타이머 창이 네이비 배경 + 라벤더 텍스트로 Settings·GiftRoom과 톤이 맞는지 확인.
3. 상태별(idle/running/completed) 버튼 표시가 색상 변경 후에도 정상인지 확인.
