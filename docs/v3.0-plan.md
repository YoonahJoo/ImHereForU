# v3.0 기획서 — 캐릭터가 책 밖으로

**작성일**: 2026-06-22
**상태**: 기획 (확정 전)
**선행 버전**: v2.0 책 UI 리워크 (완료)

---

## 1. 비전

지금까지 캐릭터는 **책(윈도우) 안**에서만 존재했다.
v3.0의 목표는 캐릭터가 **책 밖, 데스크톱 위로 나오는 것** — 화면 어디든 돌아다니며 곁에 있는 "데스크톱 메이트"로 확장한다.

> 책은 "집"이고, 캐릭터는 집 밖으로 산책 나올 수 있다.

---

## 2. 핵심 컨셉

| 항목 | v2.0 (현재) | v3.0 (목표) |
|---|---|---|
| 캐릭터 활동 범위 | 책 왼쪽 페이지(room-stage) 내부 | 데스크톱 전체 화면 |
| 윈도우 | 책 1개 (900×639) | 책 + 캐릭터 별도 레이어 |
| 상호작용 | 책 안에서 클릭/드래그 | 책 밖에서도 클릭/드래그/따라다님 |
| 존재감 | 앱을 열어야 보임 | 책을 접어도 캐릭터는 화면에 남을 수 있음 |

---

## 3. 기능 요구사항 (초안)

### 3-1. 책 밖으로 나오기
- 책 안의 캐릭터를 **밖으로 꺼내는 트리거** (예: 특정 에셋 클릭, 드래그해서 책 경계 밖으로 끌어내기)
- 나오는 순간 전환 애니메이션 (책에서 빠져나오는 연출)

### 3-2. 데스크톱에서의 행동
- 화면 위 자유 이동 (걷기/떠다니기)
- 마우스 따라다니기 또는 가장자리에 머무르기
- 클릭/드래그 인터랙션 유지 (기존 표정 시스템 재사용)
- 일정 시간 방치 시 책으로 돌아가기(귀가) 또는 졸기

### 3-3. 책과의 관계
- 책을 접어도(최소화해도) 캐릭터는 데스크톱에 남는 모드
- 캐릭터를 다시 책 안으로 넣기(귀가) 동작

## 4. 기술 과제

핵심 난제는 **"투명 윈도우 안의 캐릭터를 데스크톱 전역에서 보이게 + 그 외 영역은 클릭이 통과되게"** 하는 것이다.

| 과제 | 방향(후보) |
|---|---|
| 캐릭터를 화면 어디든 표시 | 캐릭터 전용 **별도 BrowserWindow** (transparent, frame:false, alwaysOnTop) 를 전체 화면 크기로 띄움 |
| 캐릭터 외 영역 클릭 통과 | `setIgnoreMouseEvents(true, { forward: true })` + 캐릭터 hit 영역에서만 토글 |
| 책 창 ↔ 캐릭터 창 동기화 | IPC로 상태(표정/모드/위치) 공유 |
| 멀티 모니터 / 좌표계 | `screen` API로 디스플레이 bounds 계산, 캐릭터 위치를 전역 좌표로 관리 |
| 성능 | 전체 화면 투명 윈도우 상시 렌더 → idle 시 애니메이션 최소화 |

> 기존 자산 재사용: 표정 시스템(Character), 말풍선, 표정 전환/blink/movement 애니메이션은 그대로 이식 가능.

## 5. 단계별 마일스톤 (제안)

1. **M1 — 캐릭터 전용 투명 오버레이 윈도우 PoC**
   전체 화면 투명 창에 캐릭터 1개 띄우고, 캐릭터 영역 외 클릭 통과 확인.
2. **M2 — 책 ↔ 오버레이 상태 동기화**
   IPC로 표정/모드 공유, "밖으로 나오기" 트리거 1개 연결.
3. **M3 — 데스크톱 이동/행동**
   자유 이동, 마우스 따라다니기, 방치 시 귀가.
4. **M4 — 연출 + 폴리시**
   책에서 빠져나오는/돌아가는 전환 애니메이션, 다크/라이트 대응.

## 6. 확정 필요 사항 (Open Questions)

아래는 기획 확정 전 사용자 결정이 필요한 항목:

1. **"밖으로 나오는" 트리거** — 무엇으로? (전용 버튼 / 캐릭터를 책 밖으로 드래그 / 자동)
2. **데스크톱에서의 주 행동** — 가만히 한 곳에 머묾 vs 돌아다님 vs 마우스 추적?
3. **책을 접었을 때** — 캐릭터도 같이 사라짐 vs 데스크톱에 계속 남음?
4. **범위** — 단일 모니터만? 멀티 모니터까지?
5. **v3.0의 "끝" 정의** — 어디까지 되면 v3.0 완료로 볼지?

> 위 5개가 정해지면 본 기획서를 확정본으로 갱신하고 M1부터 착수.
