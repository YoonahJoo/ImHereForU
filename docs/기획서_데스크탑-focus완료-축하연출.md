# 기획서 — 데스크탑(책 밖)에서 Focus 타이머 완료 시 축하 연출

작성일: 2026-06-30

---

## 1. 배경 / 문제

캐릭터(Yoonah)는 두 군데에 있을 수 있어요.

- **책 안** = `YoonahRoom` 화면 (메인 창)
- **책 밖(데스크탑)** = `OverlayApp` 화면 (투명 전체화면 오버레이)

Focus 타이머가 끝나면 책 안에서는 축하 연출이 나와요.

> 표정이 `smile`로 바뀌고 → 축하 말풍선이 뜨고 → 선물이 생성돼서 선물 알림 카드가 떠요.

그런데 **캐릭터가 책 밖(데스크탑)에 나와 있을 때 타이머가 끝나면**, 축하 연출 없이 곧바로 `idle` 표정으로 돌아와 버려요. 유저 입장에선 "어? 집중 다 했는데 아무 반응이 없네?" 하고 허전해요.

### 왜 이런 일이 생기나 (지금 구조)

타이머 완료 처리 함수 `handleFocusComplete()` 는 **항상 책 쪽(`YoonahRoom`)에서만** 실행돼요.
( `src/components/YoonahRoom/YoonahRoom.tsx:151` )

```
타이머 0초 도달
  → handleFocusComplete() 실행 (책 쪽)
      ├─ 책 캐릭터 표정 smile
      ├─ 축하 말풍선
      └─ 선물 생성 + 저장 + 알림 카드   ← 이건 밖에 있어도 잘 됨!
```

즉 **선물은 책 밖에 있어도 이미 정상적으로 만들어지고 저장돼요.** 문제는 그 "축하하는 모습"이 데스크탑 캐릭터에게는 전달되지 않는다는 점이에요.

데스크탑(`OverlayApp`)이 책에서 받는 타이머 정보는 **"실행 중이냐 / 아니냐(true·false)" 딱 하나**예요.
( `overlay:set-timer` 라는 신호, `OverlayApp.tsx:158` )

그래서 타이머가 끝나는 순간 상태가 `running → completed` 로 바뀌고, 책은 데스크탑에 `set-timer = false`(=실행 안 함)만 보내요. 데스크탑은 이 false를 받고 "아 끝났구나" 하며 그냥 `idle`로 돌아가는 거예요.
( `OverlayApp.tsx:71-73` )

> 비유: 책(메인 창)이 사장님이고 데스크탑(오버레이)은 직원이에요. 사장님은 직원에게 "지금 일하는 중/아닌 중"만 말해줬지, "끝났으니 축하 파티 열어!"라는 메시지는 따로 보낸 적이 없어요. 그래서 직원은 파티를 못 열었던 거죠.

---

## 2. 목표

캐릭터가 **책 밖(데스크탑)에 나와 있을 때** Focus 타이머가 끝나면, 책 안에서 했던 것과 비슷하게 **데스크탑 위에서 축하 연출**이 나오게 한다.

단, **선물창(선물 알림 카드)을 데스크탑에 그대로 띄우는 건 예쁘지 않아서 제외**한다. 대신 **선물방으로 안내하는 말풍선**으로 대체한다.

## 3. 변경 전 / 변경 후 비교

### 변경 전 (현재)

| 위치 | 타이머 완료 시 동작 |
|---|---|
| 책 안 | smile + 축하 말풍선 + 선물 생성 + 선물 알림 카드 |
| **책 밖(데스크탑)** | **곧바로 idle (아무 연출 없음)** |

### 변경 후 (목표)

| 위치 | 타이머 완료 시 동작 |
|---|---|
| 책 안 | (그대로 유지) smile + 축하 말풍선 + 선물 생성 + 선물 알림 카드 |
| **책 밖(데스크탑)** | **smile + 축하 말풍선 → 2초 뒤 선물방 안내 말풍선 → idle 복귀** |

> 선물 생성·저장 자체는 변경 전에도 정상 동작하므로 그대로 둔다. (데스크탑에선 카드만 안 띄울 뿐, 선물은 이미 선물방에 쌓여 있음)

## 4. 상세 플로우 (데스크탑, 책 밖일 때)

```
[T+0초]  타이머가 다 됨
         → 캐릭터 표정 character_smile 로 전환
         → 축하 말풍선 1 표시
            "You did a great job, sweet heart <3"  (랜덤 축하 메시지 중 하나)

[T+2초]  두 번째 말풍선 표시
            "If you wanna see what gift you got, check the gift room!"

[그 이후] 말풍선이 사라지고, 캐릭터는 idle 표정으로 복귀
```

### 타임라인 세부

- **0초**: 표정 `smile` + 축하 말풍선(말풍선은 기존처럼 약 3초간 표시).
- **2초**: 두 번째 말풍선으로 교체(선물방 안내 문구). 새 말풍선이 뜨면 이전 말풍선 타이머는 자동으로 리셋됨(기존 `showBubble` 동작 그대로).
- **idle 복귀**: 표정은 두 번째 말풍선이 충분히 보이고 난 뒤 `idle`로 돌아가게 한다. (책 안에서 smile을 5초 유지하는 것과 균형을 맞춰, 데스크탑에서도 두 번째 말풍선이 끝까지 보일 만큼 유지)

### 말풍선 카피 (확정 문구)

| 순서 | 문구 |
|---|---|
| 1번 (0초) | 기존 축하 메시지 풀에서 랜덤 (`completeMessages`) <br>예: "You did a great job, sweet heart <3" |
| 2번 (2초) | **"If you wanna see what gift you got, check the gift room!"** |

> ⚠️ 원문에 `chec` 으로 적혀 있었는데 `check` 오타로 보여요. 위처럼 **check**로 확정했어요. 괜찮으면 그대로, 다른 문구 원하면 말해 주세요.

## 5. 구현 포인트 (어디를 어떻게 바꿔야 하는가)

> ⚠️ 이 문서는 기획서이므로 아직 코드는 바꾸지 않았어요. 아래는 "구현하려면 이런 작업이 필요하다"는 설계 메모예요. 실제 수정은 별도 승인 후 진행해요.

### 핵심 아이디어

책(`YoonahRoom`)이 "타이머 완료" 순간에, **캐릭터가 밖에 있으면** 데스크탑에게 **"완료됐어, 축하 연출 시작해!"라는 새 신호**를 보내고, 데스크탑은 그 신호를 받아 연출을 재생한다.

### (1) 책 쪽 — `YoonahRoom.tsx`

- `handleFocusComplete()` ( `YoonahRoom.tsx:151` ) 안에서, **캐릭터가 밖에 나가 있는 상태(`isCharacterOut === true`)이면** 데스크탑으로 새 신호(예: `overlay:focus-complete`)를 한 번 보내준다.
  - 이때 1번 축하 말풍선에 쓸 텍스트(랜덤으로 뽑은 `completeMessages` 문구)를 함께 실어 보내면, 책과 데스크탑이 같은 문구를 쓸 수 있어 깔끔하다.
- **주의 — 순서 문제(레이스):** 현재는 타이머가 `completed`로 바뀌면 `overlay:set-timer(false)` 가 자동으로 날아가서 데스크탑이 `idle`로 돌아가요(`YoonahRoom.tsx:320-322`). 완료 신호와 이 `false` 신호가 부딪히면 축하 연출이 곧바로 idle에 덮어써질 수 있어요. 그래서 데스크탑 쪽에서 "완료 연출 중에는 set-timer(false)로 인한 idle 전환을 잠깐 무시"하도록 처리해야 해요. (아래 (2) 참고)

### (2) 데스크탑 쪽 — `OverlayApp.tsx`

- 새 IPC 신호(`overlay:focus-complete`)를 구독하는 리스너를 추가한다. (기존 IPC 구독부 `OverlayApp.tsx:144-170` 옆에)
- 신호를 받으면:
  1. `setExpression('smile')` — `character_smile` 표시
  2. `showBubble(축하문구)` — 1번 말풍선 (기존 `showBubble` 재사용, `OverlayApp.tsx:75`)
  3. 2초 뒤 `showBubble("If you wanna see what gift you got, check the gift room!")` — 2번 말풍선
  4. 적절한 시간 뒤 `idle` 복귀 (`revertSoon` 유사 패턴 재사용, `OverlayApp.tsx:82`)
- **레이스 처리:** 완료 연출이 진행되는 동안에는, `overlay:set-timer(false)`가 들어와서 표정을 `idle`로 되돌리는 `useEffect`( `OverlayApp.tsx:71-73` )가 연출을 덮어쓰지 않도록 가드를 둔다. (예: "완료 연출 중" 플래그를 두고, 연출이 끝난 뒤 idle로 정리)
- 순차 말풍선 패턴은 이미 책 쪽에 여러 군데 구현돼 있어요(`YoonahRoom.tsx:201-243`, `245-256`). 같은 `setTimeout` 방식이라 그대로 응용 가능.

### (3) 선물 관련 — 변경 없음

- 선물 생성·저장(`createGiftItem`, `saveGifts`)은 책 쪽 `handleFocusComplete`에서 이미 처리돼요. **밖에 있어도 선물은 정상 적립**돼요.
- 데스크탑에는 선물 알림 카드(`gift-notification`)를 **띄우지 않음** (의도된 기획). 대신 2번 말풍선으로 "선물방 확인" 유도.

## 6. 엣지 케이스 / 확인 사항

1. **선물방은 책 안에만 있음** — 선물방(`GiftRoom`)은 책 화면의 패널이라, 데스크탑에서 바로 못 열어요(`YoonahRoom.tsx:630`). 그래서 2번 말풍선은 "집(책)으로 돌아가서 선물방을 확인해"라는 안내 역할이에요. → 의도와 맞음.
2. **연출 도중 유저가 캐릭터를 클릭/더블클릭하면?** — 기존 데스크탑 클릭 핸들러(`handleClick` 등)가 표정/말풍선을 바꿔요. 완료 연출이 클릭으로 중단돼도 큰 문제는 없지만, "축하 도중 클릭은 무시/유지" 같은 우선순위를 정할지 결정 필요. (권장: 클릭이 들어오면 자연스럽게 클릭 반응으로 넘어가도 OK)
3. **연출 도중 🏠 버튼으로 귀가하면?** — 데스크탑이 사라지므로 연출도 종료. 책으로 돌아오면 선물은 이미 적립돼 있어 선물방에서 확인 가능. → 문제 없음.
4. **연출 도중 새 타이머를 시작하면?** — 일반적으로 완료 직후 바로 새 세션을 시작하긴 어렵지만, 만약 시작되면 `set-timer(true)`가 와서 `focus_mode`로 가야 하므로, 완료 연출 가드가 "타이머 다시 실행"에는 양보하도록 한다.
5. **smile/말풍선 유지 시간** — 책 안은 smile 5초. 데스크탑은 2번 말풍선까지 보여야 하므로 그보다 약간 길게(예: 약 5초 전후) 잡아 두 번째 말풍선이 끝까지 보이게 한다.

---

## 7. 요약

- 선물은 이미 밖에서도 잘 적립됨 → **추가 작업은 "데스크탑 축하 연출"뿐.**
- 책이 완료 순간 데스크탑에 **새 신호 1개**를 보내고, 데스크탑은 **smile → 축하 말풍선 → (2초) → 선물방 안내 말풍선 → idle** 순서로 재생한다.
- 데스크탑엔 선물 카드 대신 **선물방 안내 말풍선**으로 대체(기획 의도).
- 구현 시 핵심 주의점은 **완료 신호와 `set-timer(false)`의 충돌(레이스) 방지.**
