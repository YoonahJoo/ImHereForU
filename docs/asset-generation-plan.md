# 에셋 생성 작업 기획서 (Claude Design 버전)

---

## 1. 접근 방식 요약

### 기존 계획 (레이어 방식) → 변경

| 항목 | 기존 | 변경 |
|---|---|---|
| 방식 | body + face + eyes + arm 레이어 합성 | 표정별 완성 이미지 |
| 에셋 수 | 15개 | 12개 |
| 정렬 문제 | 레이어간 픽셀 정렬 필요 | 없음 |
| 무브먼트 | 에셋으로 처리 | CSS로 처리 |
| 예외 | — | arm_wave.png 1개만 레이어 |

### 변경 이유

AI 이미지 생성 모델은 픽셀 단위 정렬을 보장할 수 없다.
레이어 방식은 eyes/face/arm이 정확히 같은 좌표에 맞아야 하는데,
이를 생성 모델에게 요구하는 것은 근본적으로 어렵다.

완성 이미지 방식은 이 제약이 없다.
표정 변화는 같은 캐릭터에서 입/눈썹만 바꾸면 되고,
무브먼트(float, bounce, shake 등)는 CSS로 충분히 구현 가능하다.

## 3. 핵심 원칙

### 절대 기준 이미지
`yoonah.png` — 모든 에셋의 single source of truth.
새로운 캐릭터를 디자인하거나 스타일을 재해석하지 않는다.

### 캐릭터 위치 고정
12개 파일 전부에서 캐릭터의 몸, 머리, 팔이 **동일한 위치**에 있어야 한다.
표정이 바뀌어도 캐릭터가 위아래로 움직이거나 크기가 달라지면 안 된다.

### 변경 가능한 요소
- 입 모양, 입꼬리 방향
- 눈썹 각도
- 볼터치 강도
- 눈 뜸/감음
- `curious`에서만 안경 추가 허용

### 절대 변경 불가 요소
- 얼굴형, 머리 실루엣, 체형
- 눈의 크기와 위치
- 머리색, 피부톤, 의상 색상
- 선 굵기와 채색 질감
- 의상 디자인

## 4. Claude Design 전용 전략 — 2단계 접근

Claude Design에서 일관성을 높이는 가장 효과적인 방법은
**분석 먼저, 생성 나중** 구조다.

### 1단계 — 참조 이미지 분석 (매 차수 시작 시)

```
첨부한 yoonah.png를 보고 아래 특징을 구체적으로 설명해줘.
생성은 아직 하지 말고 분석만 해줘.

- 얼굴형과 머리 실루엣
- 눈의 크기, 위치, 스타일
- 입과 눈썹의 기본 형태
- 피부톤, 머리색, 의상 색상
- 선 굵기와 채색 질감
- 전체적인 그림체 특징
- 캐릭터의 캔버스 내 위치 (상단/중앙/하단 등)
```

Claude가 분석 결과를 확인해주면 2단계로 넘어간다.

### 2단계 — 생성 요청

```
방금 분석한 특징을 모두 그대로 유지하면서
[파일명]을 생성해줘.

변경할 것: [입 모양/눈썹 등 해당 표정의 변경 요소만]
변경하지 말 것: 얼굴형, 머리, 체형, 의상, 색상, 캐릭터 위치, 캔버스 크기
```

## 5. 차수별 생성 계획

### 1차 — 기준 확정 (가장 중요)

**생성 파일:** `character_idle.png`

이 파일이 모든 후속 에셋의 기준이 된다.
스타일, 색상, 캐릭터 위치가 여기서 확정된다.

**검수 포인트:**
- `yoonah.png`와 동일한 캐릭터인가
- 얼굴형, 머리 실루엣, 체형이 동일한가
- 색상(머리, 피부, 의상)이 동일한가
- 캔버스가 1110×1417인가
- 투명 배경인가

승인 조건: "이 이미지를 기준으로 나머지 11개를 뽑아도 된다"고 판단될 것

---

### 2차 — idle 변형 + arm

**생성 파일:**
- `character_idle_blink.png`
- `arm_wave.png`

**idle_blink 주의사항:**
- `character_idle.png`와 완전히 동일, 눈만 감은 상태
- 캐릭터 위치, 크기, 표정 전부 동일해야 함

**arm_wave 주의사항:**
- 팔 부위만 포함, 나머지 완전 투명
- `character_idle.png`의 팔 위치와 정확히 일치해야 함
- 손을 들어 흔드는 포즈

---

### 3차 — 고빈도 표정

**생성 파일:**
- `character_smile.png`
- `character_focus.png`
- `character_sleepy.png`

앱에서 가장 자주 표시되는 표정들. 1차 승인본을 레퍼런스로 첨부해서 요청.

---

### 4차 — 드래그 / 롱프레스 표정

**생성 파일:**
- `character_sulky_daily.png`
- `character_sulky_focus.png`
- `character_dragging_daily.png`

---

### 5차 — 나머지 표정

**생성 파일:**
- `character_dragging_focus.png`
- `character_curious.png`
- `character_cheering.png`

`curious`는 안경이 추가되므로 별도 주의. 안경이 얼굴 구조를 가리지 않아야 함.

## 6. 차수별 공통 프롬프트 구조

모든 차수에서 아래 구조를 사용한다.

```
[역할]
너는 레이어형 캐릭터 에셋을 제작하는 일러스트 작업자야.
첨부된 yoonah.png가 이 작업의 절대 기준 이미지야.

[절대 고정 조건]
아래 요소는 절대 변경하면 안 돼:
- 얼굴형, 머리 실루엣, 체형, 캐릭터 위치
- 눈의 크기와 위치
- 피부톤, 머리색, 의상 색상
- 선 굵기와 채색 질감
새로운 캐릭터를 디자인하거나 스타일을 재해석하지 마.

[기술 조건]
- 캔버스 크기: 1110×1417px
- 파일 형식: PNG, 투명 배경

[이번 차수 작업]
[파일명]을 생성해줘.
변경할 것: [해당 표정의 변경 요소]
이번 요청에서는 이 파일만 생성해줘.
```

## 7. 피해야 할 표현

- "비슷하게 맞춰줘" → 기준 불명확
- "더 귀엽게" / "더 자연스럽게" → 재해석 유발
- "나머지도 같이 만들어줘" → 차수 분리 원칙 위반
- "같은 느낌으로 새로 해석해줘" → 동일성 붕괴 직접 유발

- [ ] PNG, 투명 배경인가

### 캐릭터 동일성
- [ ] `yoonah.png`와 같은 캐릭터로 보이는가

- [ ] 캐릭터 위치가 이전 차수와 동일한가

### 표정 품질
- [ ] 표정 변화가 입/눈썹 수준의 최소 수정인가
- [ ] 감정이 읽히는가
- [ ] 표정 변화 때문에 캐릭터가 달라 보이지 않는가

### arm_wave 전용
- [ ] 팔 외 영역이 완전 투명한가
