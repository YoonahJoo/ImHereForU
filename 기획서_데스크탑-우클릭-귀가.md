# 기획서 — 데스크탑(책 밖) 캐릭터 우클릭으로 귀가 + 집 아이콘 제거

작성일: 2026-06-30 (개정)

---

## 1. 배경 / 문제

캐릭터가 책 밖(데스크탑)에 나와 있을 때, 지금은 **캐릭터 머리 위 🏠(집) 버튼**을 눌러야 책으로 돌아가요.
( 버튼: `src/overlay/OverlayApp.tsx:282` 근처 / 귀가 로직: `handleReturnHome` `OverlayApp.tsx:262` / 스타일: `src/overlay/Overlay.css:43` )

이 🏠 버튼이 항상 떠 있는 게 깔끔하지 않아서 **버튼을 없애고, 캐릭터 우클릭(오른쪽 마우스)으로 귀가**시키려고 해요. 그냥 바로 보내지 않고, 캐릭터가 한 번 되묻는 **귀여운 yes/no 확인 단계**를 넣어요.

> 이 기능은 **데스크탑(책 밖, `OverlayApp`) 전용**이에요. 책 안 화면엔 🏠 버튼도 없고 무관하므로 변경 없음.

---

## 2. 목표 (3가지)

1. **🏠 집 아이콘 제거** — 책 밖에 나왔을 때 캐릭터 위에 보이던 집 버튼을 지운다.
2. **우클릭 → 귀가 확인 플로우** — 캐릭터에 커서를 올리고 우클릭하면 "Want me to go back?" 말풍선 + [yes/no] 선택지가 뜬다.
3. **모드별로 표정/연출이 다르게**, 그리고 **focus 모드에서는 타이머·선물 이벤트와 충돌하지 않게** 처리한다.

---

## 3. 상세 플로우 (모드별)

표정 상태는 평소 기준:
- **daily 모드** 평소 표정 = `idle`
- **focus 모드(타이머 실행 중)** 평소 표정 = `focus_mode`(노트북 포즈)

우클릭으로 "Want me to go back?" 질문이 뜨는 동안에는 **표정을 평소 그대로 유지**하고, yes/no를 누른 "이후"에 표정이 바뀐다.

### 3-1. Daily 모드

```
[표정: idle]
우클릭 → 말풍선 "Want me to go back?" + [ yes ] [ no ]

 ├─ yes 클릭
 │    → 표정 character_smile + 말풍선 "Bye babe!"  (2초 유지)
 │    → 2초 뒤 책으로 귀가(pop-out 모션)
 │    → 책 안으로 들어가서 character_idle 로 복귀
 │
 └─ no 클릭
      → 표정 character_smile + 말풍선 "I knew that you wanted to be w me more hehe"  (2초 유지)
      → 2초 뒤 데스크탑에 그대로 머무르며 character_idle 로 복귀
```

### 3-2. Focus 모드 (타이머 실행 중)

```
[표정: focus_mode]
우클릭 → 말풍선 "Want me to go back?" + [ yes ] [ no ]

 ├─ yes 클릭
 │    → 표정 character_cheering + 말풍선 "Bye babe!"  (2초 유지)
 │    → 2초 뒤 책으로 귀가(pop-out 모션)
 │    → 책 안으로 들어가서 character_focus 로 복귀  (타이머 계속 흐름)
 │
 └─ no 클릭
      → 표정 character_cheering + 말풍선 "I knew that you wanted to be w me more hehe"  (2초 유지)
      → 2초 뒤 데스크탑에 그대로 머무르며 character_focus 로 복귀  (타이머 계속 흐름)
```

**타이머는 yes/no 와 무관하게 계속 흐른다.**

#### ⭐ 핵심 규칙 — 확인 플로우 도중 타이머가 다 된 경우 (선물 이벤트 보류)

위 yes/no 연출(표정 + 2초 메시지 + 귀가/복귀)이 **진행되는 도중에 타이머가 0초에 도달**하면, 선물 이벤트(축하 연출)를 **즉시 실행하지 않고 보류**한다. yes/no 플로우가 **완전히 끝난 뒤**에 선물 이벤트를 시작한다.

- **yes 였던 경우** → 캐릭터가 **책 에셋 안으로 들어간 다음**, **책 안에서 나오는 선물 이벤트**를 시작한다.
  - (책 안 버전: `character_smile` + 축하 말풍선 + 선물 알림 카드 — 기존 책 안 완료 연출)
- **no 였던 경우** → 캐릭터가 **데스크탑에 머문 상태에서**, **책 밖에서 나오는 선물 이벤트**를 시작한다.
  - (책 밖 버전: `character_smile` + 축하 말풍선 → 2초 뒤 "check the gift room!" 안내 — 직전에 구현한 데스크탑 완료 연출)

## 4. 변경 전 / 변경 후 요약

### 변경 전 (현재)
- 캐릭터 위에 🏠 버튼이 떠 있고, 클릭하면 pop-out 후 귀가.
- 우클릭은 정의된 동작 없음(OS 기본 메뉴가 뜨거나 좌클릭처럼 잘못 처리될 여지).

### 변경 후 (목표)
- 🏠 버튼 **없음**.
- 캐릭터 우클릭 → "Want me to go back?" + [yes/no].
  - daily: yes/no 모두 `smile` 연출 → `idle` 복귀.
  - focus: yes/no 모두 `cheering` 연출 → `focus_mode` 복귀, 타이머 계속.
  - focus에서 연출 도중 타이머 완료 시 선물 이벤트를 **플로우 종료 후** 실행(yes=책 안 / no=책 밖).

## 5. 말풍선 카피 (확정 문구)

| 상황 | 문구 |
|---|---|
| 우클릭 시 (질문) | **Want me to go back?** |
| yes 선택 후 | **Bye babe!** |
| no 선택 후 | **I knew that you wanted to be w me more hehe** |

> "w me" = "with me"의 캐주얼 표기로 그대로 둘게요. 다른 문구 원하면 알려주세요.

## 6. 구현 포인트 (어디를 어떻게 바꿔야 하는가)

> ⚠️ 이 문서는 기획서예요. 아직 코드는 바꾸지 않았어요. 아래는 설계 메모이고, 실제 수정은 승인 후 진행해요.

### (1) 🏠 버튼 제거 — `OverlayApp.tsx` / `Overlay.css`
- 캐릭터 위 🏠 버튼 JSX 삭제(`OverlayApp.tsx:282` 근처 `.overlay-home-btn`).
- `homeBtnRef` 및 hit-detection에서 버튼 영역 검사 제거(`OverlayApp.tsx:102-107`).
- 버튼 전용 CSS 정리(`Overlay.css:43-69`).
- **귀가 로직 `handleReturnHome`(`OverlayApp.tsx:262`)은 남겨서 yes가 재사용** — pop-out 모션 → `overlay:enter-character` → 창 숨김.

### (2) 우클릭 감지 — `Character.tsx`
- 캐릭터 래퍼(`character-wrapper`, `Character.tsx:247`)에 `onContextMenu` 추가 → `e.preventDefault()`로 OS 기본 메뉴 차단 → 새 콜백(`onRightClick`) 호출.
- **좌/우 버튼 구분:** `handleMouseDown`(`Character.tsx:223`)이 버튼 종류를 안 가리고 드래그/클릭을 시작하므로, 우클릭(`e.button === 2`)은 초반에 걸러 드래그/클릭 로직이 안 돌게 한다.
- `CharacterProps` + 콜백 ref(`cbs.current`, `Character.tsx:77`)에 `onRightClick` 추가.

### (3) yes/no 확인 UI — 신규
- 현재 `SpeechBubble`(`SpeechBubble.tsx`)은 **텍스트만** 렌더 → 버튼이 필요하므로 **`ConfirmBubble` 같은 새 컴포넌트** 권장(질문 + [yes][no]).
- 위치는 기존 말풍선 좌표(`offsetX/offsetY`) 재사용해 캐릭터 위에 띄움.

### (4) hit-detection에 yes/no 버튼 포함 — `OverlayApp.tsx` (중요)
- 오버레이는 "커서가 캐릭터 위"일 때만 클릭을 받음(`overInteractive`, `OverlayApp.tsx:92-109`). 🏠를 빼는 대신, **확인창이 떠 있는 동안 yes/no 버튼 영역을 이 검사에 포함**해야 버튼이 바탕화면으로 통과되지 않고 눌린다.

### (4-1) 확인창이 떠 있는 동안 좌클릭·드래그 차단 — `Character.tsx` (+ `OverlayApp.tsx`)
- 확인창(`confirmOpen`)이 열려 있는 동안에는 **캐릭터 좌클릭(단일/더블)·롱프레스·드래그를 전부 막는다.** (오직 yes/no 버튼만 누를 수 있음)
- 구현: `Character`에 "잠금" 상태를 내려, `handleMouseDown`(`Character.tsx:223`)에서 잠겨 있으면 드래그/클릭/롱프레스 시작 자체를 `return`으로 차단. (우클릭 자체는 위 (2)에서 별도 처리)
- yes/no를 누르면 `confirmOpen`이 풀리며 잠금 해제.

### (5) 표정·타이밍 연출 — `OverlayApp.tsx`
- 확인창 상태(예: `confirmOpen`)를 둔다. 우클릭 → `confirmOpen = true` + 질문 말풍선.
- yes/no 클릭 시 **현재 모드(타이머 실행 여부)에 따라 표정 분기**:
  - daily(타이머 미실행): `smile`, 2초 뒤 `idle`.
  - focus(타이머 실행): `cheering`, 2초 뒤 `focus_mode`.
  - (오버레이엔 이미 `restExpression()`이 타이머 상태에 맞는 평소 표정을 돌려줌 — `OverlayApp.tsx:67`)
- yes → 2초 메시지("Bye babe!") 후 `handleReturnHome()` 호출.
- no → 2초 메시지 후 그대로 머무름 + 평소 표정 복귀.

### (6) ⭐ 선물 이벤트 보류·라우팅 — focus 모드 (가장 까다로운 부분)
- 배경: 캐릭터가 밖에 있을 때 타이머가 끝나면 `handleFocusComplete`(`YoonahRoom.tsx:151`)가 **선물 데이터(생성·저장)** 와 **축하 연출**을 처리한다. 직전 작업으로, 밖이면 `overlay:focus-complete`를 보내 데스크탑 축하도 재생한다.
- 새 규칙: **확인 플로우(yes/no 연출)가 진행 중이면 축하 연출을 곧바로 띄우지 말고 보류**, 플로우가 끝난 뒤 실행.
  - **선물 데이터 생성·저장은 그대로 즉시(타이머 0초 시점) 처리해도 무방** — 보류 대상은 "시각적 축하 연출"뿐.
  - **yes**(귀가) → 캐릭터가 책에 들어간 뒤 **책 안 버전** 축하 연출을 재생.
  - **no**(머무름) → 데스크탑에서 **책 밖 버전** 축하 연출을 재생.
- 구현 메모(택1, 승인 후 상세 설계):
  - 오버레이에 "확인 플로우 진행 중" 플래그를 두고, 이 시간에 도착한 `overlay:focus-complete`(또는 완료 사실)를 **큐에 보관**.
  - 플로우 종료 콜백에서 분기:
    - no → 보관해 둔 데스크탑 축하 연출 실행.
    - yes → 귀가 완료 신호(`book:character-entered`) 이후, 책 쪽에서 책 안 축하 연출 실행. (책↔오버레이 간 "보류된 완료" 전달 방식은 구현 단계에서 확정)
- ⚠️ 주의: 선물은 **한 번만** 생성되어야 함(중복 적립/누락 방지). 보류는 연출만, 데이터는 1회.

## 7. 엣지 케이스 / 확인 사항

1. **확인창이 떠 있는 동안 좌클릭·드래그 차단(확정)** — 확인창이 열려 있으면 캐릭터 좌클릭/더블클릭/롱프레스/드래그를 모두 막는다. yes/no 버튼만 눌러서 진행/취소할 수 있다.
2. **연속 우클릭** — 확인창이 떠 있는데 또 우클릭하면 같은 창 유지(중복 생성 방지).
3. **드래그 중 우클릭** — 드래그 종료 후에만 확인창이 뜨도록.
4. **hit-detection 인셋(`HIT_INSET = 0.18`)** — 캐릭터 클릭 박스는 몸통 쪽으로 18% 줄여 잡힘(`OverlayApp.tsx:26`). yes/no 버튼은 머리 위에 뜨므로 **버튼 전용 바운드를 따로** 포함해야 눌림.
5. **focus yes + 타이머 동시 완료** — yes 2초 연출이 끝나고 귀가 → 책 안에 들어간 "뒤"에 책 안 선물 이벤트가 시작되어야 함(연출 순서 보장).
6. **no 메시지 후 복귀 타이밍** — "I knew that..." 2초 표시 후 평소 표정(daily=idle / focus=focus_mode)으로 복귀.

## 8. 한 줄 요약

🏠 버튼을 없애고 **우클릭 → "Want me to go back?" [yes/no]** 로 귀가. daily는 `smile→idle`, focus는 `cheering→focus_mode` 연출에 yes는 "Bye babe!", no는 "I knew that you wanted to be w me more hehe"(각 2초). focus에서 연출 중 타이머가 끝나면 **선물 이벤트는 플로우 종료 후**에 — yes면 책 안 버전, no면 책 밖 버전으로 실행.
